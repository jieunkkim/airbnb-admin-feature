# Airbnb 수익 분석기 (Airbnb Revenue Analyzer)

## Executive Summary

에어비앤비 호스트들이 사업자통장 거래내역을 업로드하여 자동으로 분류되고 월별/연별 수익 현황을 시각적으로 분석할 수 있는 도구입니다.

- **목표 사용자**: 수익 분석에 어려움을 겪는 에어비앤비 호스트
- **주요 가치**: 거래내역 자동분류 + 시각적 리포트 + PDF 다운로드
- **기술 스택**: HTML5 + CSS3 + Vanilla JavaScript
- **위치**: tools/airbnb-revenue-analyzer/ 폴더
- **배포**: h.run 도구 모음 index.html에 카드 추가

---

## 전체 아키텍처

사용자 흐름:

```
사용자가 도구 접속
  ↓
tools/airbnb-revenue-analyzer/index.html 로드
  ↓
사용자 입력
  ├─ 복사/붙여넣기 (raw data)
  ↓
데이터 파싱 & 정규화
  ↓
거래내역 분류
  ├─ 기본 분류 규칙 적용
  ├─ 사용자가 규칙 추가/수정/삭제
  ├─ 제외 대상 행 필터링
  ↓
분류된 데이터 브라우저 메모리 저장
  (sessionStorage)
  ↓
리포트 생성
  ├─ 월별/연별 통계 계산
  ├─ 차트 렌더링 (시각화)
  ↓
출력
  ├─ 웹 화면에 표시
  ├─ 요약 데이터 복사 기능
  └─ PDF 다운로드
```

## 데이터 모델

### 1. 거래내역 (Transaction)
- 날짜
- 거래 설명/메모
- 거래 금액
- 분류된 카테고리 (수입|월세|청소비|관리비|시설투자비|세금|소모품|기타)
- 제외 여부
- 매칭된 규칙 정보

### 2. 분류 규칙 (Classification Rule)
- 카테고리
- 키워드 (단일, 중복 불가)
- 활성화 여부
- 기본규칙 여부

매칭 방식: 거래내역의 설명이나 메모에 키워드가 포함되면 해당 카테고리로 분류 (대소문자 구분 안 함)

### 3. 리포트 데이터
- 기간별 통계 (월별/연별)
- 카테고리별 금액 및 비율 집계
- 총 수입, 총 지출, 순이익

## 기능 요구사항

### Phase 1: MVP 

#### 1.1 데이터 입력 
- 복사/붙여넣기 (raw 텍스트) 파싱 기능
- 데이터 미리보기 (테이블 형태로 확인)
- 컬럼 매핑 (사용자가 date/description/amount/memo 컬럼 지정)

#### 1.2 자동 분류 
- 기본 분류 규칙 제공 (각 카테고리별 1-2개 초기 키워드)
- 거래내역 자동 분류 로직 (규칙 기반)
- 분류 우선순위: 활성화된 규칙만 사용, 첫 매칭 규칙 적용
- 매칭되는 규칙 없으면 "기타" 분류

#### 1.3 규칙 관리 UI 
- 현재 규칙 목록 조회
  - 기본규칙: 카테고리, 키워드, 활성화 상태 표시
  - 사용자규칙: 카테고리, 키워드, 활성화 상태 표시
  
- 규칙 추가 (모달/폼)
  - 카테고리 선택 (8개 중 1개)
  - 키워드 입력 (중복 체크)
  - 활성화 상태 설정
  - "추가" 버튼
  
- 규칙 수정
  - 기본규칙도 수정 가능
  - 카테고리 변경 가능
  - 키워드 변경 (중복 체크)
  - 활성화 상태 토글
  
- 규칙 삭제
  - 기본규칙도 삭제 가능
  - 삭제 확인 팝업
  
- 키워드 중복 검증
  - 새 규칙 추가/수정 시 기존 키워드와 중복 체크
  - 중복 시 경고 메시지

#### 1.4 규칙 적용 및 재분류 
- 규칙 추가/수정/삭제 후 자동 재분류
- 분류 상태 업데이트 화면에 표시
- "규칙 초기화" 버튼 (기본규칙으로 돌아가기)

#### 1.5 제외 기능 
- 개별 거래내역 제외 체크박스
- 제외된 항목은 리포트에서 제외
- 제외 대상 임시 표시 (하단에 제외목록을 따로 표시하며 토글로 닫고 열수 있음)

#### 1.6 월별/연별 리포트 
- 전체 매출 대비 각 카테고리가 어느정도 비율을 차지하는지 보여주기 위함
- 분류된 데이터 기반 월별 통계 계산
- 연별 통계 (월별 데이터 합산)
- 카테고리별 금액 및 비율 집계
- 순이익 계산 (수입 - 지출)

#### 1.7 시각화
- 월별 추이 차트 (막대/라인차트)
- 카테고리별 파이 차트
- 주요 통계 카드 (총수입, 총지출, 순이익)
- 선택 가능한 기간 필터 (월/년)

#### 1.8 리포트 출력 
- HTML 리포트 생성 (차트 + 표 + 요약)
- 요약 데이터 텍스트 복사 기능 (클립보드 복사)
- PDF 다운로드 (html2pdf 라이브러리 사용)

#### 1.9 UI/UX
- 단계별 진행 (Input → Classification → Rules → Exclude → Report)
- 반응형 웹 디자인 (모바일 대응)
- 직관적인 네비게이션
- h.run 디자인 시스템 준수 (기존 styles.css 활용)

## 기본 분류 규칙

각 카테고리별 초기 규칙 제공 (사용자가 수정/삭제 가능):

- 수입: airbnb, 수익
- 월세: 월세, rent
- 청소비: 청소, cleaning
- 관리비: 관리, 관리비
- 시설투자비: 구매, 투자
- 세금: 세금, tax
- 소모품: 소모, 용품

키워드 중복 검증:
- 새 규칙 추가/수정 시 기존 키워드와 중복 체크
- 중복 시 경고 메시지 표시

## 개발 단계

### 필수 기능

1. **프로젝트 구조 설정**
   - tools/airbnb-revenue-analyzer/ 폴더 생성
   - HTML, CSS, JavaScript 파일 구성
   - 필요한 라이브러리 통합

2. **데이터 입력**
   - 사용자가 텍스트를 붙여넣기 할 수 있는 인터페이스
   - 입력된 데이터 파싱 및 정규화
   - 데이터 미리보기 (테이블 형태)
   - 컬럼 매핑 (사용자가 어느 열이 날짜/금액인지 지정)

3. **자동 분류**
   - 기본 분류 규칙 적용
   - 거래내역별 카테고리 자동 할당
   - 분류 결과 표시

4. **규칙 관리**
   - 규칙 목록 조회 (기본규칙, 사용자규칙 구분)
   - 규칙 추가 (카테고리, 키워드 입력)
   - 규칙 수정 (기존 규칙 변경)
   - 규칙 삭제 (기본규칙도 삭제 가능)
   - 규칙 활성화/비활성화 토글
   - 키워드 중복 여부 검증
   - 규칙 변경 시 자동 재분류

5. **제외 기능**
   - 개별 거래내역을 제외할 수 있는 체크박스
   - 제외된 항목은 리포트에서 제외됨

6. **리포트 생성**
   - 월별 통계 계산 (각 카테고리별 금액)
   - 연별 통계 계산
   - 총 수입, 총 지출, 순이익 계산
   - 카테고리별 금액 집계

7. **시각화**
   - 월별 추이 차트 (막대/라인)
   - 카테고리별 파이 차트
   - 주요 통계 카드 표시 (총수입, 총지출, 순이익)
   - 기간 필터 (월/년)

8. **리포트 출력**
   - 화면에 표시된 리포트를 HTML로 생성
   - 요약 데이터 텍스트 복사 기능
   - PDF로 다운로드 기능

9. **사용자 인터페이스**
   - 단계별 진행 화면 (Input → Classification → Rules → Exclude → Report)
   - 반응형 웹 디자인 (모바일 대응)
   - h.run 디자인 시스템과 통일성 유지

## 주의사항

## 주의사항

1. **키워드 중복 체크**: 같은 키워드가 여러 규칙에서 사용되지 않도록 검증

2. **기본 규칙도 수정/삭제 가능**: 사용자가 제공된 기본 규칙을 자유롭게 변경할 수 있음

3. **규칙은 매번 새로 시작**: 페이지 새로고침하면 기본 규칙으로 초기화됨

4. **데이터 프라이버시**: 모든 데이터는 사용자 브라우저에만 저장되고 서버로 전송되지 않음

5. **성능**: 대량의 거래내역(10,000줄 이상)도 처리 가능하도록 구현

6. **반응형**: 데스크톱, 태블릿, 모바일 모두 지원

7. **디자인 통일성**: 기존 h.run 도구들과 일관된 디자인 유지
